# Caderno Temático: Software com NotebookLM

## Contexto e objetivos

Este projeto foi desenvolvido para o Desafio 3 da DIO e apresenta um caderno temático dedicado ao estudo de Software. O objetivo é usar o NotebookLM como ferramenta de aprendizagem ativa, combinando curadoria de fontes, engenharia de prompts e registro de descobertas.

### Sobre o assunto

O tema escolhido é Software, com foco em:

- conceitos básicos e definições,
- tipos de software,
- ciclo de vida do software,
- boas práticas de desenvolvimento.

### Objetivos do estudo

- Entender o que é software e como ele é classificado.
- Revisar o ciclo de vida de desenvolvimento de software.
- Identificar boas práticas que melhoram qualidade e manutenção.
- Usar IA para gerar explicações, exemplos e revisões.
- Criar um miniguia claro para estudo e consulta posterior.

## Miniguia de estudo final

### Resumos estruturados sobre Software

- **Conceitos básicos de software**
  - Software é o conjunto de programas, dados e instruções que permite ao hardware executar tarefas.
  - Existem diferentes tipos de software conforme função, como software de sistema, software aplicativo e software embarcado.
  - Software pode ser proprietário, livre ou de código aberto.

- **Tipos de software**
  - Software de sistema: gerencia recursos do computador e fornece plataforma, por exemplo sistemas operacionais.
  - Software aplicativo: atende necessidades específicas do usuário, como editores de texto, navegadores e apps.
  - Software utilitário: auxilia tarefas de manutenção, como antivírus e ferramentas de backup.
  - Software embarcado: integra dispositivos e máquinas, como roteadores, carros e eletrodomésticos inteligentes.

- **Ciclo de vida do software**
  - Requisitos: definição do que o software deve fazer e como deve se comportar.
  - Projeto e arquitetura: planejar estrutura, módulos e interfaces.
  - Implementação: escrever código e construir o software.
  - Testes: verificar funcionalidade, desempenho e segurança.
  - Implantação: disponibilizar o software ao usuário.
  - Manutenção: corrigir problemas e adaptar o software com o tempo.

- **Boas práticas de desenvolvimento**
  - Escrever código legível e bem documentado.
  - Manter versionamento e controle de mudanças.
  - Aplicar testes automatizados sempre que possível.
  - Revisar código em pares e utilizar padrões de projeto apropriados.
  - Considerar manutenção, segurança e desempenho desde o início.

### Glossário dos principais conceitos de Software

- **Software**: conjunto de programas e instruções que controla o hardware e realiza tarefas específicas.
- **Software de sistema**: software que gerencia os recursos do computador e fornece suporte aos demais programas.
- **Software aplicativo**: programa criado para executar funções voltadas ao usuário final.
- **Ciclo de vida de software**: sequência de etapas de desenvolvimento, desde requisitos até manutenção.
- **Engenharia de software**: disciplina que aplica métodos e práticas para criar software de qualidade.
- **Manutenção de software**: aperfeiçoamento, correção e adaptação de software após sua entrega.
- **Requisitos**: necessidades e condições que o software deve atender.
- **Código limpo**: código escrito de forma clara, organizada e fácil de entender.
